# Поля карточки товара (`products.json`)

Файл каталога: **`products.json`** (в этой же папке `public/`). Каждый элемент массива — один товар.

Ниже — какие поля можно задавать и примеры. Поля **`id`**, **`name`**, **`imageUrl`**, **`category`** обязательны для отображения строки в списке; остальное по желанию.

---

## Обязательные и базовые

| Поле | Описание | Пример |
|------|----------|--------|
| `id` | Уникальный идентификатор (строка). | `"1"` |
| `name` | Название товара. | `"Canon EOS R6"` |
| `imageUrl` | Картинка: имя файла из **`public/`** или полный `https://…`. | `"testPhoto.webp"` → откроется как `/testPhoto.webp` |
| `category` | Категория (фильтр-чипы на главной). | `"Камеры и объективы"` |
| `description` | Текст под заголовком в раскрытой карточке. | `"Зеркальная камера…"` |
| `article` | Артикул на упаковке; показывается рядом с категорией. | `"DT-003"` |

---

## PDF (`pdfs`)

Массив инструкций из **`public/`**. Можно список строк или объекты с подписью.

**Вариант 1 — только имя файла:**

```json
"pdfs": [ "manual.pdf", "warranty.pdf" ]
```

**Вариант 2 — подпись вторая строка (`meta`):**

```json
"pdfs": [
  { "file": "manual.pdf", "meta": "24 стр." },
  { "file": "quick-start.pdf", "meta": "2 стр." }
]
```

По нажатию открывается просмотр PDF в модальном окне.

---

## Видео (`videos`)

Так же, как PDF: файлы лежат в **`public/`**, в JSON указываете только имя файла. Загрузка ролика по сети — **после** нажатия на строку (не при открытии каталога).

```json
"videos": [
  "promo.mp4",
  { "file": "unboxing.mp4", "meta": "3:40" }
]
```

Поддерживаются обычные веб-форматы (например **`.mp4`**).

---

## Внешние ссылки (`links`)

Массив объектов. У каждой ссылки:

| Поле | Обязательно | Описание |
|------|----------------|----------|
| `label` | да | Текст строки в карточке. |
| `href` | да | URL (`https://…`, `http://…`, `tg://…`). |
| `meta` | нет | Вторая строка мелким текстом (домен, длительность и т.п.). |
| `openIn` | нет | **`browser`** (по умолчанию) или **`telegram`** — см. ниже. |

### Ссылка в браузере (`openIn`: `"browser"` или не указывать)

Открывается в **новой вкладке** (`target="_blank"`).

```json
"links": [
  {
    "label": "Официальный сайт",
    "href": "https://www.example.com/",
    "meta": "example.com",
    "openIn": "browser"
  }
]
```

`"openIn": "browser"` можно **не писать** — поведение то же самое.

### Ссылка в Telegram (`openIn`: `"telegram"`)

Нужно, чтобы по возможности открылось **приложение Telegram**, а не только вкладка браузера.

- Для ссылок вида **`https://t.me/…`** или **`https://telegram.me/…`** приложение подставит схему **`tg://`** (например, пользователь / канал: `tg://resolve?domain=…`, приглашение: `tg://join?invite=…`).
- Если уже указан **`tg://…`**, он передаётся как есть.

```json
"links": [
  {
    "label": "Наш канал",
    "href": "https://t.me/your_channel",
    "meta": "t.me",
    "openIn": "telegram"
  },
  {
    "label": "Приглашение в чат",
    "href": "https://t.me/+AbCdEfGhIjKlMnOp",
    "meta": "приватная ссылка",
    "openIn": "telegram"
  }
]
```

**Важно:** на десктопе без Telegram клиента ОС может показать диалог или ничего не сделать — это ограничение окружения, а не сайта. На телефоне с установленным Telegram обычно открывается приложение.

### Дубликаты

Одна и та же пара **`openIn` + `href`** в одном товаре не покажется дважды.

## Прямая ссылка на PDF или видео (адресная строка)

При открытии просмотра PDF или видео с главной в URL добавляются параметры — можно **скопировать ссылку** и отправить покупателю: откроется тот же файл (если он есть у этого товара в `products.json`).

Формат (всё на главной `/`):

| Параметр | Значение |
|----------|----------|
| `open` | `pdf` или `video` |
| `p` | **`id` товара** из `products.json` (строка). |
| `f` | **имя файла** в `public/` (как в `pdfs` / `videos`), без ведущего `/`. При необходимости кодируйте в URL (`encodeURIComponent`). |

**Примеры:**

- PDF `test.pdf` у товара с `"id": "3"`:
  - `/?open=pdf&p=3&f=test.pdf`
- Видео с длинным именем (фрагмент закодирован):
  - `/?open=video&p=18&f=vecteezy_sunrise-and-sunset-timelapse-high-definition-footage-on_3213023.mp4`

Если товар не найден или у него **нет** такого файла в списке, параметры из URL убираются, модалка не открывается.

Кнопка «Назад» в браузере после открытия файла обычно сначала убирает эти параметры и закрывает просмотр.

---

## Полный минимальный пример товара

```json
{
  "id": "101",
  "name": "Пример товара",
  "imageUrl": "testPhoto.webp",
  "category": "Аксессуары",
  "description": "Краткое описание для покупателя.",
  "article": "SKU-101",
  "pdfs": [ { "file": "test.pdf", "meta": "Инструкция" } ],
  "videos": [ { "file": "demo.mp4", "meta": "Обзор" } ],
  "links": [
    { "label": "Сайт", "href": "https://example.com", "meta": "example.com", "openIn": "browser" },
    { "label": "Поддержка в TG", "href": "https://t.me/support", "openIn": "telegram" }
  ]
}
```

После правок **`products.json`** сохраните файл и обновите страницу в браузере (при `npm run dev` данные подхватываются с сервера разработки).
